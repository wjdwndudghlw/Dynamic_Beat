주요키워드: JFRAME, THRED

JFRAME 을 상속했다고 Paint 함수 연속실행 x 

JFrame을 상속받은 클래스에서 paint 함수의 동작 방식은 자동적으로 "계속 실행되는" 것이 아니라, 특정 이벤트에 의해 트리거되는 방식입니다.
JFrame과 같은 AWT/Swing 컴포넌트의 그리기 동작은 시스템이나 사용자의 상호작용에 의해 필요할 때 호출됩니다.

paint 함수의 호출 시점
1. 초기 윈도우 생성 시: 컴포넌트가 처음 화면에 나타날 때 paint 메소드가 호출됩니다.
2. 사이즈 변경 시: 윈도우 크기가 조정되거나 최소화/최대화 되었다가 원래 크기로 돌아올 때.
3. 다른 윈도우에 가려졌다가 다시 나타날 때: 다른 윈도우나 프로그램에 의해 가려졌다가 다시 보여질 때.
4. 운영 체제나 사용자에 의한 명시적인 요청: 사용자가 윈도우를 드래그하여 다른 위치로 이동시키거나, 운영 체제가 다시 그리도록 요청할 때.
5. repaint() 메소드 호출 시: 프로그래밍적으로 repaint() 메소드를 호출하여 강제로 다시 그리기를 요청할 수 있습니다.

THRED 상속했다고 RUN 함수 연속 실행 X 
THRED를 상속한 클래스 선언 후 Start 함수를 사용하면 RUN 메소드가 실행되면 RUN메소드 내 WHILE(TRUE)를 사용하여 무한정 SCREENDRAW 라는 메소드 호출이 가능하다.
